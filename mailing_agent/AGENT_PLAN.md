# 📋 에이전트 5요소 정의 — 메일링 에이전트 AI

> **버전:** 1.0.0
> **최종 업데이트:** 2026-06-16

---

## 🎯 1. 목표 (Goal)

**사용자의 자연어 요청을 받아 메일을 작성하고, 안전 검증 후 Mock 발송 결과를 제공한다.**

- 사용자가 입력한 요청을 분석하여 수신자, 제목, 본문을 추출
- 발송 전 미리보기 제공 + 사용자 승인 필수
- 모든 과정에서 안전 규칙(가드레일)을 적용하여 위험 입력 차단

## 🔧 4. 도구 (Tools)

| 도구명 | 등급 | 입력 | 출력 | 설명 |
|--------|------|------|------|------|
| `composeMail` | 🟢 자동 | to, subject, body | 성공 여부 | 메일 내용 포맷팅 → 채팅 출력 |
| `showPreview` | 🟢 자동 | to, subject, body | 성공 여부 + draft | 카드 UI + [확인]/[취소] 버튼 |
| `sendMail` | 🔴 승인 | (draft, 내부 호출) | 성공 여부 + messageId | Mock 발송 실행 |

### 도구 등급 기준

| 등급 | 조건 | 예 |
|------|------|-----|
| 🟢 자동 (auto) | 외부에 영향이 없거나 UI 내에서만 동작 | composeMail, showPreview |
| 🔴 승인 (ask) | 외부(실제 발송 등)에 영향이 있음 | sendMail |

## 🛡️ 안전 체계 요약

| 가드레일 | 검사 시점 | 위반 시 |
|----------|----------|--------|
| 도구 허용 목록 | 모든 도구 호출 시 | 즉시 차단 + 사유 출력 |
| 수신자 검증 | sendMail 진입 전 | 거부 + 친구 목록 안내 |
| sendMail 권한 | sendMail 실행 시 | write 외 동작 차단 |
| 입력 검증 | 모든 사용자 입력 | 거부 + 500자 이내 요청 |
| 콘텐츠 안전 4종 | 모든 사용자 입력 | 거부 + 3요소 메시지 (거부+사유+대안) |
| 미리보기 승인 게이트 | sendMail 실행 전 | showPreview [확인] 필요 안내 |
| 재시도 제한 | 도구 실패 시 | 최대 5회 + 사용자 선택권 |
| 비상 정지 | 항상 (버튼) | 즉시 모든 작업 중단 |
